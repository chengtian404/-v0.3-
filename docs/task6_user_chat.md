# 任务 6 / 6.3：用户侧 AI 对话与智能问数

## 完成范围

### 登录注册闭环

- 普通用户注册直接写入后台 `users` 表，角色固定为 `Normal User`。
- 后台禁用或删除账号后，用户侧立即失效。
- 普通用户只能进入用户侧；`System Admin` 只能进入后台。
- 密码由服务端使用 PBKDF2-SHA256 和独立随机盐保存。
- 兼容旧版前端 SHA256 后再提交的存量普通用户账号。

### 对话工作台

- A 区：LOGO 与系统名称。
- B 区：模型下拉选择，优先选中后台默认模型。
- C 区：会话任务列表，首条消息自动生成标题，可读取历史和删除。
- D 区：用户/AI 双侧气泡、数据库问数、报表和 API 数据卡片。
- E 区：输入框、发送、`/` 快捷菜单和 `@` 数字员工菜单。
- 桌面端与移动端均支持响应式侧栏和稳定布局。

### 意图识别与调度

意图路由只输出白名单类型：

1. `general_chat`：调用用户选择模型或后台默认模型。
2. `database_query`：调用安全问数工具，只返回文本指标。
3. `data_report`：调用安全问数工具，并生成 ECharts 报表数据。
4. `digital_employee`：`@名称` 调度后台启用的数字员工。

LLM 型数字员工优先使用自身绑定模型；未绑定时回退后台默认模型。API 型数字员工返回数据卡片。

### 数据库问数

已开放的只读统计主题：

- 用户数量、状态、角色、注册趋势。
- 数据仓库总量、来源、深度采集状态、入库趋势。
- 瞭源数量与启停状态。
- 模型数量、类型和 Token 使用量。
- 数字员工数量、类型和启停状态。
- 系统核心数据概览。

所有主题由固定后端查询实现。用户文本只决定主题和展示方式，不能成为 SQL 的任何组成部分。

### 报表

- 支持柱状图、折线图和饼图。
- 仅当意图包含“图表、报表、趋势图、分布图、可视化”等明确要求时返回 `report_json`。
- 普通问答和普通数据库问数不会附带报表。

## 安全限制

- 拒绝 SQL 代码块和 SELECT/INSERT/UPDATE/DELETE/DROP/ALTER 等 SQL 语句模式。
- 拒绝覆盖系统规则、泄露系统提示词或开发者消息的提示词注入模式。
- 模型接口不会获得数据库连接、SQL 或 API Key。
- 会话按 `user_id` 校验归属，用户不能读取或删除他人的会话。
- 数字员工公开列表不返回模型配置、API 地址、Headers 或认证信息。
- API 数字员工限制 HTTP/HTTPS、请求方法、响应大小，并启用 TLS 证书验证。
