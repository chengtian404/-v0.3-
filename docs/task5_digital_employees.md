# Task 5: 数字员工模块开发

## 完成时间
2026-07-14

## 概述
数字员工模块支持两种类型的数字化员工能力实现。该系统允许后台管理员创建、配置和管理数字员工，这些员工可以通过 @mention 方式被调度参与任务执行。

## 功能模块

### 数字员工管理
- 列表查看：支持分页（20条/页）、搜索、按类型筛选
- 新增：支持输入名称、描述、头像(Emoji)、员工类型
- 编辑：修改数字员工的所有配置
- 删除：移除数字员工
- 启用/禁用：控制数字员工是否可被调用
- 执行：在线测试数字员工的响应

### 类型1: LLM型数字员工
基于大语言模型的员工，核心配置：
- **关联模型**: 可选择模型引擎中已配置的模型，或使用系统默认模型
- **系统提示词 (System Prompt)**: 定义员工角色和行为
- **技能 (Skills)**: 可选扩展能力（JSON格式描述）
- **Crawl4AI**: 可选网页爬取能力
- **Temperature / Max Tokens**: 模型参数配置

### 类型2: API型数字员工
基于HTTP/HTTPS接口的员工，核心配置：
- **API URL**: 接口地址，支持 {{input}} 占位符
- **请求方法**: GET / POST / PUT / DELETE
- **请求头 (Headers)**: JSON格式自定义请求头
- **查询参数 (Params)**: JSON格式URL参数
- **请求体模板**: POST请求的Body模板，支持 {{input}} 占位符
- **响应格式**: JSON / 纯文本
- **API认证**: 无认证 / Bearer Token / Basic Auth
