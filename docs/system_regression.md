## 本次交互修复

- 数字员工页面改用 `{% raw %}` 输出员工和模型 JSON，避免配置模型后脚本被 HTML 转义。
- 数字员工 API URL 与 Body 中的 `{{input}}` 以字面量显示，不再被 Tornado 解析为内建函数。
- 管理员会话改用 `admin_username` Cookie，用户侧继续使用 `username`，前后台可在同一浏览器中同时登录。
- 遗留的管理员 `username` Cookie 会在访问用户登录页时自动清理，点击“返回用户侧”不再跳回后台。
- 数字员工和模型表单改用浏览器原生 `<dialog>`，不依赖 Layui CDN 即可打开、关闭和重复打开。
- 接入模型与数字员工弹窗尺寸根据浏览器窗口自适应。
- 模型分类、启用状态、数字员工类型、关联模型、API 请求方法、响应格式和认证类型下拉框均已纳入交互回归。
- 模型表单新增 DeepSeek 官方预设，自动填写官方 Base URL、`deepseek-chat` 和 Provider。
- API Key 误填 URL、DeepSeek 空密钥会在浏览器与服务端双重拦截。
- 模型测试保留上游错误正文，并对常见 HTTP 状态返回中文诊断信息。
- 流式请求不再强制发送 `stream_options`，提升 DeepSeek 和中转服务兼容性。
